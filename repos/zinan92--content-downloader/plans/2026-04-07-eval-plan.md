# Eval Plan — content-downloader

**Date:** 2026-04-07
**Target:** zinan92/content-downloader
**Repo type:** skill (capability adapter)

## What This Repo Claims

- **Claim 1 — One URL in, standardized files out.** 给任意支持的 URL，输出统一的 `media/` + `metadata.json` + `content_item.json` + `text.txt` + `manifest.jsonl`。
- **Claim 2 — 4 平台真实可用。** Douyin（含 XBogus 签名 + Playwright fallback）、Xiaohongshu（XHS-Downloader sidecar 自管理）、WeChat OA（HTML 解析）、X/Twitter（yt-dlp）。
- **Claim 3 — Failure transparency。** Unsupported / auth-required / sidecar-down 等场景都给清晰错误，不会假装成功。
- **Claim 4 — Dedup + idempotent。** 已下载的 content_id 自动跳过。

## What We Will Validate

- **Scenario 1 — Douyin 单视频（核心路径）**：API 签名失败时 Playwright fallback 是否真的能补救。
- **Scenario 2 — WeChat OA 文章**：下游能不能拿到非空的 article.html / 标题 / 文本。
- **Scenario 3 — X 推文**：能不能拿到媒体或文本。
- **Scenario 4 — 不支持平台的 URL**：是否清晰报错并列出 supported list。
- **Scenario 5 — 重复下载**：第二次跑同一个 URL 是否跳过。

> **未覆盖（已知 gap）：** Xiaohongshu、Douyin profile 批量、Cookie fetcher、CI 测试套件复跑。本次只覆盖单 URL 直跑路径。

## Real Inputs We Will Use

- **Input A** — Douyin: `https://www.douyin.com/video/7621048932151414054`（无 cookies，强制走 fallback 路径）
- **Input B** — WeChat OA: 两个真实的 `mp.weixin.qq.com/s/*` URL（验证不是单 URL 偶发）
- **Input C** — X: `https://x.com/elonmusk/status/1849889982030823521`
- **Input D** — Bad URL: `https://example.com/foo`
- **Input E** — Repeatability: 重跑 Input A

## How Many Times We Will Test

- **Core path（Douyin 视频）：** 2 次（首跑 + 重跑验证 dedup）
- **Variation path（WeChat × 2 + X × 1 + bad URL × 1）：** 4 次
- **Repeatability check：** 通过 dedup 重跑覆盖

## What Counts As Passing

每个场景必须同时满足：
1. CLI 退出码 0（错误场景退出码非 0）
2. 标准化输出文件存在 **且非空**（不允许 0-byte 文件）
3. `content_item.json` 关键字段（title / author / media_files OR text content）非空
4. CLI 报告的 "Downloaded" 状态与磁盘上的产物一致（不允许"假装成功"）
5. 失败场景的错误信息可操作（告诉用户下一步做什么）

## If Everything Passes, What We Can Trust

- **Minimum trust level：** `reusable` — 可以让下游 pipeline 直接消费 ContentItem 而不用每次手工检查。
- **Stretch trust level：** `recommendable` — 仅当再补 XHS、Douyin profile 批量、长期稳定性测试后。
- **Remaining risk：**
  - 平台 API 变化导致 adapter 失效
  - Cookie 过期处理是否优雅
  - sidecar 进程崩溃恢复

# TradeCat — static-checks run, 2026-05-04

Eval scope: install path, dataset registry, CLI surface, TUI fallback,
and auto-update policy. No live install. End-to-end sync deferred to a
molecule-level live run.

## Findings

### claim-001: install.sh — **passed**

The install script is a real, well-shaped POSIX shell script (not a
curl-baited stub). Five environment overrides are exposed for shape
control:

```
TRADECAT_INSTALL_REPO    → repo URL override
TRADECAT_INSTALL_BRANCH  → branch (default develop)
TRADECAT_INSTALL_DIR     → app dir (default ~/.tradecat/app)
TRADECAT_BIN_DIR         → bin dir (default ~/.local/bin)
TRADECAT_PYTHON_VERSION  → Python version (default 3.12)
```

Plus two CI-friendly skip flags (`TRADECAT_INSTALL_SKIP_SYNC`,
`TRADECAT_INSTALL_SKIP_PATH_WRITE`) — both are referenced by name in
the script body, matching the README documentation. Pre-flight checks
gate on `python_ok`, fall back to `uv` if no system Python 3.12 exists.

### claim-002: dataset registry — **passed**

4 datasets, all `active=true`, modes match the README's snapshot vs
stream split:

| Dataset | Mode | GID |
|---|---|---|
| market_snapshot | snapshot | 1904613219 |
| anomaly_panel | snapshot | 1915220137 |
| market_stats | snapshot | 1161752788 |
| event_stream | stream | 1419246950 |

Note: each dataset is a Google Sheets tab GID under one of two
workbook keys (`market_data`, `alternative_data`). This is a genuine
single-source-of-truth move — README, install path, and one-shot
request.py all share the same registry file. The trade-off is total
dependence on Google Sheets availability and on the project owners
not renaming/moving tabs (see `watch_out` in repo.yaml).

### claim-003: Python version consistency — **passed**

`install.sh` defaults `PYTHON_VERSION=3.12`. `pyproject.toml` declares
`requires-python = ">=3.12"`. All three documented command names —
`tradecat`, `tradecat-terminal`, `tcat` — exist in
`[project.scripts]`. No version drift, no missing entry points.

### claim-004: zero-install request.py — **passed**

`scripts/request.py` is 7468 bytes (real script, not placeholder),
references `dataset_registry.json` plus the documented `--datasets`,
`--meta`, `--headers` flags. The README's claim that this script
"shares the same dataset contract" with the installed CLI holds —
both read the same registry file.

### claim-005: TUI graceful fallback — **passed**

`src/tradecat_terminal/tui.py` carries 22 references to the documented
fallback env vars and `--plain` mode (`TRADECAT_TERMINAL_FORCE_CURSES`,
`ALLOW_WINDOWS_CURSES`, `ALLOW_SSH_CURSES`, plain). README's promise
that browser-web/SSH/Windows terminals get static-text fallback
without traceback has structural code support.

### claim-006: auto-update with throttle + override — **passed**

`install.sh` has 8 references to the documented auto-update env vars
(`TRADECAT_NO_AUTO_UPDATE`, `TRADECAT_FORCE_UPDATE`,
`TRADECAT_UPDATE_INTERVAL_SECONDS`). The launcher is generated by the
install script, so the throttle behavior is at install time pinned to
what the script writes — verified statically.

## What is still untested (claim-007)

The end-to-end live sync — `tradecat init && tradecat sync &&
tradecat` — has not been run on this machine. To promote past
`usable`:

1. Run the curl|sh installer in a clean shell.
2. Run `tradecat sync` and confirm `.tradecat/cache/snapshots/*.json`
   contains real (non-empty) JSON.
3. Run `tradecat tui --plain` and confirm at least one dataset row
   renders. (Use `--plain` so the eval doesn't depend on a curses
   environment.)
4. Re-run `tradecat sync` and confirm cache update doesn't error on
   already-synced datasets (idempotency check).
5. Log results under `runs/<date>/run-live-sync/business-notes.md`
   with: install exit code, cache file sizes, row counts per dataset,
   exit code of TUI.

## Verdict implication

Static layer is unusually clean — README is honest about what it
doesn't do, install path is well-engineered, and code structure
matches every documented surface. No false promises found.

Per the framework's molecule rule, the project still cannot be
promoted past `usable` without at least one logged live-sync run,
because the actual user value (seeing real data) is downstream of the
network + Google Sheets dependency that no static check can validate.

Recommended bucket: **usable** (with a clear path to `reusable` once
live sync is logged).

# Eval Plan — visual-explainer

**Date:** 2026-04-07
**Target:** nicobailon/visual-explainer @ `9a97a58` (main)
**Repo type:** skill (Claude Code skill + reference templates + install/share scripts)
**Framework version:** repo-evals @ `21f90eb` (provenance + hybrid-repo cap rule)

## What This Repo Claims

- **Claim 1 — 把 ASCII 表格 / ascii art 升级成 styled HTML**：8 个 slash command（generate-web-diagram / diff-review / plan-review / project-recap / fact-check / generate-visual-plan / generate-slides / share）+ 自动 trigger（4+ rows or 3+ cols 的表格）。
- **Claim 2 — Self-contained HTML，不需要 build step**：4 个参考模板，inline CSS + ESM imports for Mermaid/Chart.js，浏览器直开。
- **Claim 3 — 多 host 支持**：Claude Code marketplace / Pi (`install-pi.sh`) / OpenAI Codex（手工 cp）三种安装路径。
- **Claim 4 — Anti-AI-slop 美学契约**：constrained aesthetics（Blueprint / Editorial / Paper / Mono Terminal）+ 显式禁止 neon dashboard、gradient mesh、Inter+violet。
- **Claim 5 — Share 到 Vercel**：`share.sh` 一行命令拿到 live URL。

## What We Will Validate

这是一个 hybrid repo（同 frontend-slides）—— 真正的用户价值在 8 个 LLM-driven 命令产出的 HTML 质量，但 LLM 端只能在真实 Claude Code 会话里验证。本次评测覆盖 **support layer**：

**A 层 — 可执行 / 可静态验证的部分（本次范围）**
- **A1**：marketplace.json + plugin.json 结构合法，source path 存在，**版本号一致**
- **A2**：SKILL.md 引用的 references/templates/commands 全部存在
- **A3**：4 个 HTML 模板在 headless Chromium 里渲染，0 page errors
- **A4**：`install-pi.sh` 在干净环境里跑通（占位符替换 + prompts 复制）
- **A5**：`share.sh` 失败模式有清晰错误 + 退出码 1
- **A6**：8 个 commands/*.md 文件跟 SKILL.md 表格一致

**B 层 — LLM-driven 部分（本次不覆盖，hybrid cap 适用）**
- **B1**：在 Claude Code 里跑 `/visual-explainer:generate-web-diagram`、`/diff-review`、`/plan-review` 等命令，验证产物 HTML 质量、aesthetic 多样性、是否避开 SKILL.md 列出的 AI slop 模式
- **B2**：用一个会触发 Proactive Table Rendering 的提示，看是否自动 trigger HTML

## Real Inputs We Will Use

- **Input A1** — `.claude-plugin/marketplace.json` + `plugins/visual-explainer/.claude-plugin/plugin.json`（静态 JSON parse + 版本对比）
- **Input A2** — SKILL.md grep markdown 链接 → 文件存在性检查
- **Input A3** — 4 个 HTML templates → headless Chromium with CDN blocked + screenshot
- **Input A4** — `HOME=/tmp/ve-install-test bash install-pi.sh`（沙箱 HOME，先 mkdir parent dir）
- **Input A5** — `share.sh` × 3 个失败场景（无参数 / 文件不存在 / vercel-deploy skill 不存在）
- **Input A6** — `ls commands/` vs `grep -E '^\| `' SKILL.md` 对齐

## How Many Times We Will Test

- **Static checks（A1, A2, A6）**：一次性 grep / parse
- **Template render（A3）**：4 次（4 个模板 × 1 次 headless 渲染）
- **Install script（A4）**：先没 mkdir 跑一次（验证 mkdir bug），再 mkdir 后跑一次（验证 happy path）
- **Failure modes（A5）**：3 次
- **B 层：** 完全跳过，按 hybrid-repo cap rule 上限锁在 `usable`

## What Counts As Passing

A 层每个场景必须：
1. 退出码符合预期（成功 0，失败场景 ≥ 1）
2. JSON / HTML / Markdown 结构有效
3. 文档承诺的功能在代码里真的实现了
4. README / SKILL.md 内部引用与文件系统一致
5. **版本号在所有清单文件之间一致**

B 层不参与本次过线判断，但根据 hybrid-repo cap rule **整体 verdict 上限 = `usable`**。

## If Everything Passes, What We Can Trust

- **Minimum trust level（本次范围）：** `usable` — support layer 全过 = 安装、模板、失败模式都不会坑用户，但 LLM 端没验证 → 整体上限 `usable`
- **Stretch trust level：** `reusable` — 必须再补一次 B 层会话评测：在 Claude Code 里跑 ≥3 个不同命令，对照 SKILL.md 美学契约打分
- **Remaining risk：**
  - LLM 是否真的遵守 anti-AI-slop 规则（不写 neon dashboard / gradient mesh / Inter+violet）
  - LLM 是否真的会读 references/ 而不是凭记忆写
  - Proactive table rendering 触发是否可靠
  - share.sh 跟 vercel-deploy skill 的真实交互未测

# Eval Plan — zinan92/intel (api-service)

FastAPI + React self-hosted market-intelligence pipeline. Evaluated
against a live local instance at `http://localhost:8001` running commit
3c6eaa9, with the same commit cloned fresh to /tmp/intel for code review.

## What This Repo Claims

### Endpoints (README "API Endpoints")

- **claim-002** — Core Data: `/api/health`, `/api/articles/latest`,
  `/api/articles/search`, `/api/articles/digest`, `/api/articles/signals`,
  `/api/articles/sources`
- **claim-002** — Frontend Read Model: `/api/ui/feed`, `/api/ui/items/{id}`,
  `/api/ui/topics`, `/api/ui/sources`, `/api/ui/search`
- **claim-002** — Events: `/api/events/active`, `/api/events/{id}`,
  `/api/events/history`
- **claim-002** — Users: `/api/users`, `/api/users/{username}` (write — not smoked)

### Non-endpoint claims

- **claim-001** — Core sources work out of the box with zero API keys
- **claim-003** — FastAPI error semantics (404 / 422) on the documented surface
- **claim-004** — Deterministic responses (no random ordering in hot path)
- **claim-005** — Optional sources degrade gracefully when credentials absent
- **claim-006** — 290+ tests passing (README test badge)
- **claim-007** — `/health` honestly surfaces per-source status
- **claim-008** — Event aggregator actually produces ranked events

### Explicitly NOT claimed (skipped)

- **Auth model** — README doesn't claim auth. This is a self-hosted service.
- **Rate limits** — README doesn't claim rate limits.

## What We Will Validate

### Live layer (primary evidence)

- **claim-002** — `curl` every documented endpoint, assert HTTP 200 +
  non-empty structured JSON; detail endpoints hit with a real ID pulled
  from the list endpoints
- **claim-003** — `curl` a missing-but-valid-type ID (expect 404) and a
  malformed query (expect 422)
- **claim-004** — same endpoint twice, `md5sum` the responses, expect match
- **claim-005** — `/api/health` shows `no_data` for optional sources
  without configured credentials (social_kol, observed)
- **claim-007** — `/api/health` shows honest per-source `status` with
  `last_collected` timestamp

### Test layer (supporting)

- **claim-006** — run `pytest tests/` against the cloned repo,
  record pass/fail counts

### Source inspection (supporting)

- **claim-001** — read `collectors/` to confirm the 6 keyless source types
  are wired in; cross-check against `/api/health` counts

## How Many Times We Will Test

- Live endpoints: each endpoint hit once, 2 endpoints hit twice for
  determinism check
- pytest: one full run

## What Counts As Passing

- Live: every documented endpoint HTTP 200 with non-empty body
- Error semantics: 404 on missing resource (valid type), 422 on type error
- Test suite: ≥ 95% pass rate OR failures scoped away from endpoints
- Optional sources: `no_data`/`stale` statuses visible, not fabricated `ok`

## If Everything Passes, What We Can Trust

- Minimum trust: `reusable` — one evaluator, one machine, one moment in time
- Would take us to `recommendable`: second independent environment
  (fresh VM / different OS) verifies the same endpoint surface; the 2
  failing pytest cases get fixed; the stale reddit collector gets fixed
  or the `stale` status auto-heals.

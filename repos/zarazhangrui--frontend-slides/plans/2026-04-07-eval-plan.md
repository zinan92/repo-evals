# Eval Plan — frontend-slides

**Date:** 2026-04-07
**Target:** zarazhangrui/frontend-slides
**Repo type:** skill (Claude Code skill + supporting scripts)

## What This Repo Claims

- **Claim 1 — 一个 Claude Code skill 帮非设计师做出"不像 AI slop"的 HTML 演示文稿**：12 个 curated style preset，"show, don't tell" 的视觉发现流程。
- **Claim 2 — 零依赖产物**：单 HTML 文件，所有 CSS/JS 内联，10 年后还能打开。
- **Claim 3 — Viewport fitting 是 NON-NEGOTIABLE**：每张 slide 必须 fit 100vh，不允许 scroll，强制 clamp() 字号。
- **Claim 4 — PPT 转 web**：`extract-pptx.py` 抽 .pptx 的文字+图片+notes 成结构化 JSON。
- **Claim 5 — 一键导出 PDF / 部署到 Vercel**：`export-pdf.sh` + `deploy.sh` 提供最后一公里。
- **Claim 6 — 可装为 Claude Code 插件**：通过 `/plugin marketplace add zarazhangrui/frontend-slides` 安装。

## What We Will Validate

由于这是一个 LLM-driven skill（Phase 1-3 是给 Claude 的 prompt，不是可执行代码），评测必须分两层：

**A 层 — 可执行/可验证的部分：**
- **Scenario A1**：Plugin marketplace + plugin.json 结构正确，所有引用文件存在
- **Scenario A2**：SKILL.md 引用的所有 supporting files 真实存在
- **Scenario A3**：`extract-pptx.py` 在真实 PPT 上跑通，输出 JSON + 图片
- **Scenario A4**：`export-pdf.sh` 把一个 3-slide HTML 转成 3 页 PDF
- **Scenario A5**：失败模式（无 .slide / 文件不存在）有清晰错误 + 退出码 1
- **Scenario A6**：STYLE_PRESETS.md 包含 README 承诺的 12 个 preset

**B 层 — LLM-driven 部分（本次 eval 不覆盖，需要单独跑 Claude Code 会话）：**
- **Scenario B1**：给个话题让 skill 走完 Phase 1-5，验证产物 viewport-fit / 风格独特 / 内容准确
- **Scenario B2**：把 extract-pptx 输出喂给 Phase 2-5，验证 PPT 内容真的转成了对应 HTML

## Real Inputs We Will Use

- **Input A1** — 真实 .pptx：`/Users/wendy/Documents/New project/How To Be Successful - Sam Altman.pptx`（15 slides，多图）
- **Input A2** — 第二个真实 .pptx：`/Users/wendy/work/content-co/binary-choices/binary-choices.pptx`（8 slides，混合）
- **Input A3** — 一个 minimal 3-slide HTML deck（自建）
- **Input A4** — 一个没有 .slide 的 HTML（无效 deck）
- **Input A5** — 一个不存在的 HTML 路径

## How Many Times We Will Test

- **Core path（extract + export-pdf）：** 3 次（2 PPT + 1 HTML）
- **Failure path：** 2 次（无 .slide + 不存在文件）
- **Structure path：** 一次性 grep / file existence 检查
- **Repeatability：** 不在本次范围（脚本是无状态的）

## What Counts As Passing

A 层每个场景必须：
1. 退出码 0（错误场景必须非 0）
2. 输出文件存在且非空 / 是 valid format（PDF magic number、PNG header、JSON parse）
3. 错误信息可操作（告诉用户下一步做什么）
4. 文档承诺的功能在代码里真的实现了

B 层不参与本次过线判断，但作为 verdict 的天花板限制（无端到端验证 → 不能 `recommendable`）。

## If Everything Passes, What We Can Trust

- **Minimum trust level：** `reusable` — 可以安装这个 skill 作为日常工具，可以信任 extract + export 脚本进入 pipeline。
- **Stretch trust level：** `recommendable` — 必须再补一次端到端 LLM session 验证："给个 topic → 跑出 viewport-fit + 风格独特的 deck"。
- **Remaining risk：**
  - LLM 是否真的遵守 viewport-fitting non-negotiable 规则
  - 12 preset 在不同 mood × 不同内容下的实际产出质量
  - extract-pptx 对带 SmartArt / 嵌入视频 / 复杂表格的 PPT 行为未知
  - deploy.sh 与 Vercel 的实际交互未验证

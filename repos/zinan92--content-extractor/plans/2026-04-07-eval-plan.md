# Eval Plan — content-extractor

**Date:** 2026-04-07
**Target:** zinan92/content-extractor
**Repo type:** skill (multimodal extractor + LLM 分析)

## What This Repo Claims

- **Claim 1 — 多模态提取**：4 个 adapter (video / image / article / gallery)，把 content-downloader 的输出转成 transcript + analysis + structured_text。
- **Claim 2 — 裸文件直接喂**：`.mp4 / .mov / .mkv / ... / .mp3 / .wav / .m4a / .aac / .ogg / .flac` 都能直接传 path，CLI 自动包装为 ContentItem。
- **Claim 3 — 失败透明**：6 种 fail 场景（unsupported / invalid / hallucination / gallery / LLM / batch）都有定义的行为。
- **Claim 4 — 测试质量**：README badge 写 221 tests / 96% coverage。
- **Claim 5 — 幂等批量**：`extract-batch` 自动 skip 已提取，错误隔离。

## What We Will Validate

- **Scenario 1**：真实 Douyin 视频 ContentItem → 完整三件套输出，验证 transcript 段落、analysis 字段、structured_text 可读性。
- **Scenario 2**：上游 wechat_oa 写出的 0 字节 article.html → 验证降级行为（不崩溃 + 标记空）。
- **Scenario 3**：裸 .mp3 文件 → 验证 README 列出的音频扩展名是否真的支持。
- **Scenario 4**：不存在路径 / 无效 JSON / 不支持的 content_type → 退出码 + 错误信息。
- **Scenario 5**：已提取目录的二次 batch 跑 → 验证幂等。
- **Scenario 6**：`pytest --collect-only` 验证 README badge 的 221 tests 数字。
- **Scenario 7**：`grep` whisper 配置 → 验证 Mac 上是否真的能利用 Apple Silicon。

## Real Inputs We Will Use

- **Input A** — 真实 Douyin 视频 ContentItem (170MB MP4，山姆奥特曼成功学话题，中文)
  - 因为 Mac 上 CPU-only 转录极慢，用 ffmpeg 截前 30 秒后再跑
- **Input B** — 真实 wechat_oa ContentItem (上游 0 字节 article.html，来自 content-downloader eval)
- **Input C** — 真实 4.6MB .mp3 (videocut 项目里的播客片段)
- **Input D** — `/tmp/nonexistent-blah`
- **Input E** — `/tmp/ce-bad/content_item.json` 写非法 JSON
- **Input F** — Input A 的同一目录二跑

## How Many Times We Will Test

- **Core path：** 视频 1 次（30s 短 clip）
- **Article 边界：** 1 次（空 HTML）
- **Audio 路径：** 1 次（.mp3 裸文件）
- **失败模式：** 3 次（nonexistent / bad JSON / unsupported audio）
- **Idempotency：** 1 次（batch 二跑）
- **静态检查：** 测试 collect + transcribe.py device 配置

## What Counts As Passing

每个场景必须：
1. 退出码符合预期（成功 0，失败 ≥1）
2. 输出文件存在 **且非空**（degenerate case 例外，但要在 stdout 显示状态）
3. README 承诺的输入类型必须真的能处理
4. README 写的数字（test 数 / coverage）必须能复现
5. 错误信息可操作

## If Everything Passes, What We Can Trust

- **Minimum trust level：** `reusable` — 可以在 trading-co 的 content pipeline 里上线
- **Stretch trust level：** `recommendable` — 需要再补 image / gallery adapter 验证 + 多种语言/口音的 hallucination 检测
- **Remaining risk：**
  - faster-whisper 在不同时长/不同口音下的稳定性
  - LLM 分析对极短/极长 transcript 的鲁棒性
  - Image / gallery adapter 这次完全没碰

# goose-skills eval plan — 2026-05-04

> Archetype: **prompt-skill** (catalog of prompt skills). Layer:
> **molecule** at the repo level (collection wired by manifest +
> installer). Each individual skill has its own per-skill layer:
> capabilities = atoms, composites = molecules, playbooks = compounds.

## What this repo claims

From `README.md` + `skills-index.json` + `package.json`:

- 108 GTM / sales / marketing / SEO / CI / lead-gen skills (README
  headline) — actually 204 in the manifest
- 3 categories: 51 Capabilities + 52 Composites + 5 Playbooks (README) /
  143 + 56 + 5 (manifest)
- One-liner install: `npx gooseworks install --claude/--cursor/--codex/--all`
- Each skill ships SKILL.md + skill.meta.json with installation contract
- Skill `packs` (e.g. lead-gen-devtools 7-skill toolkit) bundle related
  skills with shared `.env.example` + `requirements.txt`
- Works with Claude Code, Cursor, and Codex

## What we cannot validate now (claim-007)

Live skill execution in real Claude Code / Cursor / Codex. The
catalog's instructional content is markdown — its quality only
manifests when an LLM actually reads SKILL.md and acts on it.
Deferred to a molecule-level scenario run with at least one example
per category (capability, composite, playbook).

- 001..006: each documented surface present in the manifest with
  numbers ≥ what the README claims (or honestly disclosed if README
  count is stale)
- 007 (deferred): a skill from each category produces a useful
  artefact in a real agent session, with logged session evidence

## If everything passes

- Static-only: `usable` — the install path, the contract, and the
  taxonomy are real and consistent
- + 1 live execution per category: still capped at `usable` because
  per-skill quality varies and the sample is small
- + ≥10 logged skill executions across categories with mostly
  positive verdicts: candidate for `reusable`
- `recommendable` requires a clear story for skill-level quality
  (not just catalog-level "we have 204") and probably some kind of
  per-skill verdict registry

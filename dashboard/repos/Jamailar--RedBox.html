Layer-aware evaluation

Layer: declared compound. Multiple molecules; call graph decided at runtime by an LLM or human. Higher layers must show evidence at every lower level too — see docs/LAYERS.md.

Atom-level evals applies (lower-level dependency)

Single responsibility. Same input → same output. No skill-level callouts.

input_contractDoes the atom reject malformed input with a clear, actionable error?
output_contractDoes the output shape match what the README/SKILL.md claims?
determinismSame input → same output within stated tolerance. (LLM atoms: structurally equivalent, not byte-equal.)
idempotenceRe-running with the same input produces the same observable end state.
no_skill_calloutsDoes the atom avoid invoking other documented skills? Primitives only (LLM API, stdlib, OS).
failure_mode_clarityDoes each documented failure mode produce a distinct, recognisable error?

Molecule-level evals applies (lower-level dependency)

2–10 atoms wired by predefined orchestration. No runtime LLM routing.

workflow_correctnessDoes an end-to-end happy path run from initial input to final output?
declared_call_graphAre the atoms the molecule documents as using actually the ones it calls? No undocumented hidden dependencies.
stop_conditionsDo documented stop conditions (success, max-retries, timeout) trigger and halt cleanly?
handoff_pointsWhen the molecule should hand back to a human, does it do so at the documented trigger?
atom_evidenceDoes every declared atom dependency have a passing atom-level eval (referenced or co-located)?
error_propagationDoes a downstream atom failure surface at the molecule boundary with enough info to act?
partial_failure_handlingWhen one atom fails, does the molecule avoid silently producing an incomplete 'success'?

Compound-level evals applies (this layer)

Multiple molecules; call graph decided at runtime by an LLM or human.

goal_achievementIn N real scenarios, does the system reach a useful end state — not just 'ran without error'?
direction_judgmentWhen the goal is ambiguous, does the system ask, pick a reasonable default, or wedge?
quality_judgmentDoes the system stop when output is good enough, instead of overshooting or stopping too early?
meaningful_autonomyWhen left to drive itself, does it actually make progress, or does it spin / loop / hallucinate?
handoff_timingWhen the system asks for help, is that the right moment?
observed_call_graphWas the call graph recorded during each scenario? Does each sub-molecule called have a passing molecule-level eval?
failure_recoveryWhen a sub-call fails, does the LLM adapt (try alternative, ask user) or wedge?

Compound experiments human-driven

The call graph is decided at runtime, so compound eval is scenario-driven. Run each scenario, observe the behaviour, log a verdict. Capture which sub-molecules actually got called — they each need their own passing molecule-level eval.
